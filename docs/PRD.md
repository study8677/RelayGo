# RelayGo 中转站收集、比价与评测平台 PRD

**版本：** V0.2  
**目标市场：** 中国大陆  
**默认货币：** 人民币（CNY）  
**阶段：** MVP

## 1. 产品定位

RelayGo 是面向中国大陆 AI API 用户的中转站信息、模型比价与评测平台。平台把各站分散的模型、费率、充值、支付、发票、企业服务和稳定性信息标准化，让个人开发者与企业采购人员能够搜索、筛选、比较与验证中转服务。

平台不代售额度、不托管资金、不担保站点经营，也不允许邀请码、返利链接或优惠码。未认领站点仍可公开收录；“已认领”仅代表运营关系通过验证，不代表平台背书。

## 2. 用户问题

- 同一模型在哪个中转站的实际成本最低？
- 倍率、充值兑换率与输入/输出单价如何换算？
- 站点是否稳定，模型是否与标称型号基本一致？
- 是否支持支付宝、微信支付、退款、对公转账与开发票？
- 是否可开专票、签合同、提供账单、子账号与 SLA？
- 价格、模型、域名或服务状态最近发生了什么变化？

## 3. 产品目标与指标

### 3.1 目标

1. 建立中转站与标准模型信息库。
2. 建立透明、可追溯的统一价格口径。
3. 提供模型、价格、支付、开票、企业服务和状态的组合筛选。
4. 提供 2～4 个站点横向比较。
5. 后期由平台承担费用，进行公共 API 可用性、真实性与能力检测。
6. 后期通过爬虫采集公开价格、公告、状态和发票政策。
7. 通过清晰标记的广告、站长会员和数据服务商业化。

### 3.2 MVP 指标

| 指标 | 目标 |
| --- | ---: |
| 收录中转站 | ≥ 100 |
| 标准化模型 | ≥ 50 |
| 完整费率覆盖率 | ≥ 70% |
| 企业服务信息覆盖率 | ≥ 50% |
| 七日内更新数据 | ≥ 80% |
| 对比功能使用率 | ≥ 15% |

## 4. 用户与权限

### 4.1 访客（MVP）

无需账号即可搜索、筛选、查看详情、比较站点以及匿名提交新站和纠错。匿名提交需要验证码、限流与人工审核。

### 4.2 注册用户（后续）

收藏、保存筛选、订阅变动、提交评价、使用自有 API Key 私有检测并管理检测历史。

### 4.3 站长（后续）

可通过域名邮箱、DNS TXT、指定文件、Meta 标签或材料审核认领站点，维护公开信息、申请复测和购买商业服务；不得修改平台检测、删除历史价格、隐藏风险或影响自然排名。

### 4.4 管理员

维护站点、模型映射、价格、企业信息、审核、风险、检测预算和广告。

## 5. MVP 功能

### 5.1 首页与中转站目录

- 全局搜索、关键指标和热门模型入口。
- 卡片展示支持模型、最低有效价格、支付方式、发票、状态、延迟、可信度与更新时间。
- 筛选：模型、价格、支付方式、可开票、专票、对公转账、合同、认证状态及可用状态。
- 排序：综合推荐、价格、可用率、延迟、模型数量和更新时间。
- 广告位与自然结果严格分离并标记。

### 5.2 中转站详情

- 基础资料、域名、API 协议、数据来源、认领状态及风险。
- 模型表：标准模型名、站内别名、输入/输出价格、倍率、上下文、能力和更新时间。
- 支付、最低充值、兑换率、退款、余额退回。
- 发票、企业服务、可用性、真实性、能力分和检测记录。

### 5.3 发票与企业服务

记录是否开票、普票/专票、数电票/电子/纸质、发票内容、开票主体、门槛、周期、按充值或消费开票、申请方式、红冲/重开、邮费和验证时间。

记录对公转账、合同、企业认证、月结、子账号、部门/项目额度、账单与明细导出、专属客服、SLA、私有线路及固定出口 IP。

运营、收款、开票主体分开记录，并展示主体是否一致。主体可信度分为未知、站长声明、公开核验和材料核验。

### 5.4 统一价格

保留原始货币、单位、输入/输出/缓存价格、倍率、充值兑换率、适用条件、来源、生效和采集时间。

```text
站内消耗 = 基准模型价格 × 模型倍率
人民币有效成本 = 站内消耗 ÷ 充值兑换率
典型调用成本 = 输入量 × 有效输入单价 + 输出量 × 有效输出单价 + 附加费用
```

默认按普通用户无需附加条件可获得的价格排名；条件价、会员价与套餐价独立展示。缺失值不得按零处理，估算必须展示公式、汇率来源与更新时间。

### 5.5 对比中心

一次比较 2～4 个站点，统一货币和 Token 单位，对比模型、有效价格、充值、支付、发票、企业服务、可用率、延迟、真实性、能力分、风险和更新时间。筛选与对比状态可通过 URL 分享。

### 5.6 提交与纠错

接受新站、域名、模型、价格、支付、发票、企业服务、故障与停运变更。提交需附来源或证据，经过“待审核 → 审核中 → 通过/拒绝 → 发布”，并保留变更历史。禁止邀请码、返利、优惠码、广告及隐私泄露。

## 6. 后期检测系统

### 6.1 公共检测

平台使用自有或站点授权的凭证并承担费用，检测连通性、模型列表、对话、流式输出、JSON、工具调用、多模态、上下文、限流、首 Token 延迟、总延迟和输出速度。

按站点和模型设置日预算、Token 上限及平台月度总预算；高频探活使用短请求，完整评测低频执行，失败采用指数退避。站长可赞助检测额度但不能影响结果。

### 6.2 模型真实性与一致性

综合响应模型名、元数据、上下文窗口、工具格式、多模态、特征问题、输出风格、拒答行为和多次测试分布，给出“高度一致、基本一致、存在差异、疑似动态路由、数据不足、检测失败”。不得凭单次结果绝对断言真假。

### 6.3 能力评测

覆盖数学、代码、中文、知识、指令遵循、结构化输出、工具调用、长上下文、多轮一致性、视觉、安全和稳定性。展示题集版本、参数、样本量、均值、波动和基线差异；结合公开与隐藏动态题，防止题库污染。

## 7. 自动采集（后续）

采集公开首页、价格页、文档、公告、状态页、模型列表、发票、支付、退款、服务条款和隐私政策。流程为“采集 → 解析 → 标准化 → 差异比较 → 生成候选 → 自动或人工审核 → 保存历史”。价格大幅变化、域名/主体/退款/发票政策变化必须人工审核。

## 8. 评分与商业化

客观分由可用率、成功率、性能、价格竞争力、透明度、完整度、新鲜度及企业服务构成；真实性、能力、口碑和风险单独展示。评分必须可解释。

广告必须标记且不改变自然排序；严重风险站点暂停投放。站长会员可获得增强资料、数据同步、分析与报告导出，但不能修改检测、评分、风险或历史。

## 9. 数据与安全

核心实体包括 `Station`、`StationEntity`、`StationEndpoint`、`Model`、`StationModel`、`PricingPlan`、`PaymentPolicy`、`InvoicePolicy`、`EnterpriseService`、`ProbeResult`、`BenchmarkRun` 和 `Submission`。

- 管理后台使用 MFA 与审计日志。
- API Key 不写入日志，私有检测默认不持久化明文 Key。
- 检测系统防范 SSRF、DNS Rebinding、恶意重定向和超大流式响应，禁止内网、环回与云元数据地址。
- 发票、合同和主体材料只供审核并脱敏，禁止公开个人敏感信息。
- 每项关键数据均展示来源、时间、验证状态与计算口径。

## 10. 路线图

### P0

目录、详情、标准模型映射、费率换算、支付和发票企业信息、筛选、2～4 站对比、匿名纠错、管理录入、历史、风险、基础网页状态与独立广告位。

### P1

自动价格爬虫、站点认领、主体核验、平台公共 API 检测、监控、真实性检测、能力评测、站长会员和企业报告。

### P2

账号、收藏订阅、私有检测、评价、多地区节点、公共数据 API、Webhook、团队成本管理和浏览器插件。

## 11. MVP 验收

- 用户能通过关键词、模型、价格、支付、开票和企业条件筛选站点。
- 用户能在详情中查看来源明确的模型、费率、支付、发票和服务信息。
- 有效价格计算口径一致，缺失数据不显示为零。
- 用户可选择 2～4 个站点并打开统一维度对比。
- 未认领站点可以展示且认领状态明确。
- 广告有明确标识且不影响自然结果。
- 匿名提交经过校验与审核，禁止邀请码、返利和优惠码。
